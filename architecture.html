🎤 YouTube 卡拉 OK — 系統架構

React PWA + 本地 Node backend，三裝置即時協作：手機遙控 / 筆電控制台 / 電視全螢幕。 支援 Spotify 公開歌單匯入、demucs ML 去人聲、PWA app mode 全螢幕。

三個角色

📱 PHONE（任何瀏覽器）

沙發遙控器；同網域 WiFi
  • http://<LAN_IP>:5174/remote
  • 搜尋 YT（CC 偵測標記）
  • 歌單 / 插播 / 重排 / 立即播放
  • 導唱值 chip（樂觀 UI）
  • ⏪ -10s / ⏯ / ⏩ +10s / 🔄 重頭 / ⏭ 切歌
  • 📺 舞台 / 📋 下一首側欄
  • 📚 過去歌單載入
  • 🎶 Spotify 公開歌單匯入 wizard
  • 🎵 lrclib 重抓歌詞
  • 看 demucs % + ETA + ready 慶祝

💻 LAPTOP DASHBOARD（控制台 + 出聲）

React PWA, vite dev: 5173 / 桌面捷徑開為 standalone app mode
  • PlaybackScreen + Web Audio worklets
  • AudioContext singleton（vocal / pitch / mixer）
  • 3 個 audio elements：A=原曲、B=demucs 伴奏、C=demucs 人聲
  • localStorage 6 個 stores（settings/queue/history/played/recent）
  • 是「唯一播放權威」：應用所有命令
  • 裝 PWA 後 window.open 自動 app mode（無 chrome）

📺 STAGE WINDOW（拖到 TV）

獨立 popup；PWA mode 自動無 chrome
  • /stage.html（同源、static）
  • video(muted) + 字幕 + demucs HUD
  • 右側側欄：下一首 + 接下來 3-5 首 + prefetch 狀態
  • 結尾 30s 自動跳出側欄
  • F11 / ⛶ 真．全螢幕；PWA 模式不需要
  • 主畫面廣播 → 立刻同步

本地 Stores（localStorage 持久化）

useSettings

導唱值、pitch、bufferSize、demucsModel、自動 demucs 開關、麥克風來源、mic alignment 等

useQueue

當前歌單 items + prefetch 狀態 map

useQueueHistory

命名 snapshots + auto_latest（每次歌單變動 5s debounce 自動覆寫）

usePlayedHistory

所有唱過的歌（cap 500），含 playCount、lastPlayedTs；pickRandom Fisher-Yates 給隨機推薦

useRecent

最近播放（cap 10），給空狀態頁卡片用

Web Audio 圖（dashboard tab）

3 個 audio element 各對一個 GainNode（同 element 永遠對同一個 slot —— 避免 Web Audio 禁止重複 createMediaElementSource 的限制）。Track-A 走 vocalNode 做三頻段去人聲； Track-B/C 是 demucs 已分離乾淨的軌，直接 bypass 接 pitchNode。 

audioA (原曲/普通)  ─► sourceA ─► gainA ─► vocalNode ───┐
                                       (三頻段)           ├─► pitchNode ─► mixerNode ─► destination
audioB (demucs 伴奏)  ─► sourceB ─► gainB ───────────────────┤           (雙讀頭          ▲
                                                       │            delay-line)      │
audioC (demucs 人聲)  ─► sourceC ─► gainC ───────────────────┘                            │
                                                                                          │ mic input 1
micStream ─► micSource ──────────────────────────────────────────────────────────────────┘

demucsMode === false （三頻段模式）：
  gainA = 1, gainB = 0, gainC = 0
  guideVocalLevel → vocalNode ("set-guide-vocal")

demucsMode === true （demucs 雙軌模式）：
  gainA = 0（靜音，但仍是主時鐘），gainB = 1（伴奏滿），gainC = guideVocalLevel（人聲）
  → 真・KTV 導唱：0% 純伴奏，100% 原曲，30% 跟唱

典型一首歌時間軸

T+0:00使用者貼 YT URLparseId → fetchQuick
T+0:01backendyt-dlp --dump-single-json（meta + captions + 偵測原文 lang）
T+0:10dashboardaudio m4a → audioA.play()，三頻段可唱；usePlayedHistory.recordPlay
T+0:10並行HEAD /api/youtube/instrumental 觸發 demucs；fetch /visual
T+0:30visual 完成video element 接上、currentTime sync 到 audio（stage 開啟時 pause）
T+1:30demucs 完成同時產生 ${id}.nv-htdemucs.m4a 與 ${id}.vox-htdemucs.m4a
T+1:30dashboardaudioB.src + audioC.src → canplaythrough → crossfadeToDemucs(250ms)
之後導唱 chip切到任意 % → gainC = guideVocalLevel（60ms ramp）
歌單背景QueuePrefetcher 一首一首 HEAD instrumental → 排隊歌也都 cache 好
queue 變動5s 後useQueueHistory.autoSave → 「📌 上一次的歌單」snapshot 永遠最新

Spotify 匯入時間軸

T+0:00使用者貼 spotify 分享連結parseSpotifyPlaylistId 抽 22 字元 ID
T+0:01backendfetch https://open.spotify.com/embed/playlist/<id>
T+0:03解析parse __NEXT_DATA__ JSON，遞迴 walk 找 trackList
T+0:03回前端跳出 checkbox 列表，預設前 30 勾選
T+30s使用者勾完POST /api/spotify/match-yt 含 tracks 陣列
T+30sbackendmapWithLimit(3) 平行對每首跑 ytdlpSearch + 評分
T+1:00-2:00結果30 首約 1-2 分鐘，回 [{videoId, title, hasCaptions}...]
T+2:00確認使用者按「全部加入歌單」→ sendCmd("enqueueBatch") 一次送 → dashboard for-loop enqueue
T+2:00背景QueuePrefetcher 自動排隊跑 demucs，第一首唱完前後面幾首已 ready

主要 endpoints

YouTube 三階段

GET /api/youtube/quick?url=階段 1：5-15 秒回 audioUrl + captionsUrl + visualUrl + captionTracks + 自動挑 lang
GET /api/youtube/visual?id=階段 2：video-only mp4，背景拉
GET /api/youtube/instrumental?id=&model=階段 3a：demucs 伴奏 m4a
GET /api/youtube/vocals?id=&model=階段 3b：demucs 人聲 m4a（與 instrumental 同 job）
GET /api/youtube/search?q=yt-dlp ytsearchN + 平行 CC 偵測
GET /api/youtube/progress?id=&model=demucs 即時進度 + ETA
GET /api/youtube/captions?id=&lang=YouTube CC（vtt），多語 fallback

🎶 Spotify 公開歌單

GET /api/spotify/playlist?url=embed scraping 抓歌單 trackList（無需 credentials）
POST /api/spotify/match-yt餵 tracks[]，每首跑 ytsearch + 評分挑 best 回 [{videoId, hasCaptions}...]

手機遙控

GET /remote送 server/remote.html
POST /api/remote/statedashboard 推狀態（含 snapshots + upcoming + hqProgress）
GET /api/remote/statephone 拉狀態
POST /api/remote/commandphone 送指令（enqueueBatch / loadSnapshot / guide / skip / sidebarToggle...）
GET /api/remote/commandsdashboard 取走指令
GET /api/remote/infoLAN URLs（QR 用）

暫存檔管理

GET /api/cache/infocache 大小 / 檔數 / 路徑
POST /api/cache/clear一鍵清空

關鍵設計取捨

  • 單一播放權威：dashboard 是唯一寫者；phone 只送命令、看狀態 → 不會有衝突
  • 3 個 audio elements 永久綁定 3 個 slot：避免 Web Audio 對同元素二次 createMediaElementSource 報錯
  • demucs 共用 job：HEAD /instrumental 觸發分離；GET /vocals 走同一個 inflight promise
  • stale callback guard：切歌時用 currentVideoIdRef 標記，上一首 demucs 跑完 的 callback resume 時偵測歌已換 → 放棄套用
  • 樂觀 UI（手機）：按下 → 立即金色脈動 → server 確認後實心 → 3s 沒回確認則 revert
  • 舞台用新視窗 + BroadcastChannel：dashboard CSS-hide 無法繞 OS 視窗邊界， 獨立 popup 可全螢幕
  • PWA install 是真正的全螢幕解：安裝後 window.open 自動 app mode（無 chrome）， 完全繞過 requestFullscreen API 的 user activation 限制
  • GPU 算力門檻：VRAM < 3GB 或 CC < 5.0 → demucs 強制走 CPU（避免 OOM 或 swap thrash）
  • Cache 持久化 + 跳出 OneDrive + 避開 CJK 路徑：CACHE_DIR = ~/.karaoke-cache （demucs Python 在 Windows 對 CJK 路徑超脆弱，必須避開）
  • Spotify embed scraping > Web API：API 自從 2024-11 起需要 owner Premium， embed 頁面內嵌的 __NEXT_DATA__ JSON 含完整 trackList，無需 credentials
  • auto-save queue debounce 5s：每次歌單變動寫 auto_latest snapshot， 瀏覽器當掉/不小心清空都救得回來

掉過的坑

  • requestAnimationFrame 在背景 tab 被凍結（Chrome 88+）→ demucs 完成 callback 用 rAF 等 frame 就永遠不跑 → 改成同步 crossfade
  • 切歌時 setVideoId(null) 廣播 null 到 stage → stage 清空 video.src 顯示 placeholder → guard if (!videoId && !title) return
  • position handler 只在 paused 狀態變化時 v.play() → 若 v.src 還在 load 失敗就再也不 retry → 改成「每次都校正」+ canplay handler 自動 play
  • BroadcastChannel onmessage handler 在 [] effect 裡 closure 抓 stale state → broadcastDataRef 鏡像最新 state
  • Dashboard + stage 兩個 video element 同解 H.264 → GPU 雙倍負擔 + demucs 搶資源 → stage 開啟時 dashboard video.pause()
  • YouTube auto-translate 把英文歌翻成爛中文 CC → detectOriginalLang 偵測原文 + pickBestCaptionLang 只用原文 manual / auto，絕不 fallback 到 auto 翻譯版
